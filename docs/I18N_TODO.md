# i18n — доработки после добавления локалей

Список задач для доведения мультиязычности после коммита `feat(i18n): add Thai and main locales`.

---

## Обязательно

- [ ] **Полные переводы UI для de, fr, es, it, pt, th**  
  Сейчас новые локали используют английские строки из `en` с переведённым только блоком `settings`. Нужно заполнить в `frontend/src/i18n/translations.ts` объекты `de`, `fr`, `es`, `it`, `pt`, `th` полными переводами (аналогично `ru`/`en`). Можно делать по одному языку.

- [ ] **Тесты: NavigationContainer**  
  В тестах `DashboardScreen` и `ChatScreen` падает «Couldn't find a navigation object». Оборачивать рендер экранов в `NavigationContainer` (и при необходимости в провайдер темы/локали), чтобы `useFocusEffect` и навигация работали в тестах.

- [ ] **Проверка сценариев**  
  Прогнать вручную: смена языка в профиле и в меню дашборда, сохранение в профиле, заголовок `X-App-Language` в запросах, ответы ИИ (чат, питание, оркестратор, аналитика) и пуш-уведомления на выбранном языке.

---

## По желанию

- [ ] **Дополнительные языки**  
  По той же схеме ([ADDING_NEW_LANGUAGE.md](ADDING_NEW_LANGUAGE.md)) можно добавить: ja (японский), zh (китайский), pl (польский), nl (нидерландский).

- [ ] **Тайский: полный словарь**  
  Отдельно доперевести объект `th` в `translations.ts`, чтобы весь интерфейс был на тайском, а не на английском.

- [ ] **Даты и числа**  
  Проверить все экраны, где форматируются даты/числа, и при необходимости передавать `locale` в форматтеры (сейчас расширен только `formatNavDate` в DashboardScreen).
